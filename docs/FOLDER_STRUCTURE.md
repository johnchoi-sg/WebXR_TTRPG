# Folder Structure

## Overview

The project is organized into clear, logical folders for better maintainability.

```
WebXR_TTRPG/
├── src/                           # Source Code
│   ├── main.js                   # Main application logic (Three.js, WebXR)
│   ├── config.js                 # Configuration (Variant Launch toggle)
│   └── style.css                 # Application styles
│
├── public/                        # Static Files
│   └── index.html                # HTML entry point
│
├── docs/                          # Documentation
│   ├── README.md                 # Full documentation
│   ├── CONFIG_GUIDE.md           # Configuration guide
│   ├── DEPLOY.md                 # Deployment instructions
│   ├── IOS_SETUP.md              # iOS-specific setup
│   ├── ANDROID_SETUP.md          # Android-specific setup
│   ├── VARIANT_LAUNCH_INTEGRATION.md  # Variant Launch details
│   ├── QUICK_FIX.md              # Quick troubleshooting
│   └── FOLDER_STRUCTURE.md       # This file
│
├── .github/workflows/             # CI/CD
│   └── deploy.yml                # GitHub Actions deployment
│
├── dist/                          # Build Output (generated, not in git)
│   ├── index.html                # Built HTML
│   └── assets/                   # Built JS and CSS
│
├── node_modules/                  # Dependencies (not in git)
│
├── .gitignore                     # Git ignore rules
├── package.json                   # NPM dependencies and scripts
├── package-lock.json              # Locked dependency versions
├── vite.config.js                 # Vite build configuration
└── README.md                      # Project overview
```

## Folder Purposes

### `/src` - Source Code
Contains all application source code:
- **`main.js`** - Main Three.js application, WebXR logic, AR functionality
- **`config.js`** - Configuration options (Variant Launch toggle, API keys)
- **`style.css`** - All CSS styles for the UI

**When to edit:**
- Adding new features
- Modifying AR behavior
- Changing UI styles
- Toggling iOS support

### `/public` - Static Files
Contains files served directly without processing:
- **`index.html`** - HTML entry point, references src files

**When to edit:**
- Changing HTML structure
- Adding meta tags
- Modifying page title

### `/docs` - Documentation
All documentation files:
- Setup guides
- Configuration instructions
- Deployment procedures
- Troubleshooting

**When to edit:**
- Updating documentation
- Adding new guides
- Documenting features

### `/dist` - Build Output
Generated by `npm run build`. Contains production-ready files.

**DO NOT EDIT MANUALLY** - This folder is automatically generated.

### `/.github/workflows` - CI/CD
GitHub Actions workflows for automated deployment.

**When to edit:**
- Changing deployment process
- Adding CI/CD steps
- Modifying build pipeline

## File Paths

### In Development
When running `npm run dev`:
- Root: `http://localhost:3000/`
- Vite serves from `public/` folder
- Imports resolved automatically

### In Production
After `npm run build`:
- Files bundled to `dist/`
- Base path: `/WebXR_TTRPG/` (GitHub Pages)
- All imports bundled into single JS file

## Import Paths

### In HTML (`public/index.html`)
```html
<link rel="stylesheet" href="../src/style.css">
<script type="module" src="../src/main.js"></script>
```

### In JavaScript (`src/main.js`)
```javascript
import * as THREE from 'three';
import { config } from './config.js';
```

## Adding New Files

### New Source File
1. Create in `src/` folder
2. Import in `main.js` or other source files
3. Vite will bundle automatically

### New Static Asset
1. Add to `public/` folder
2. Reference with relative path in HTML

### New Documentation
1. Create in `docs/` folder
2. Link from main README.md

## Build Process

1. **Development:**
   ```bash
   npm run dev
   ```
   - Serves from `public/`
   - Hot module replacement
   - Source maps enabled

2. **Production:**
   ```bash
   npm run build
   ```
   - Bundles all source files
   - Minifies JS and CSS
   - Outputs to `dist/`
   - Applies base path for GitHub Pages

3. **Preview Production Build:**
   ```bash
   npm run preview
   ```
   - Serves `dist/` folder locally
   - Test before deploying

## Git Workflow

### Tracked Files
- All source code (`src/`)
- Static files (`public/`)
- Documentation (`docs/`)
- Configuration files (root)
- GitHub Actions (`.github/`)

### Ignored Files (`.gitignore`)
- `node_modules/` - Dependencies
- `dist/` - Build output
- IDE files
- OS files

### Deployment
1. Push to `main` branch
2. GitHub Action runs automatically
3. Builds project (`npm run build`)
4. Deploys `dist/` to GitHub Pages

## Best Practices

1. **Source Code** - Always edit files in `src/`, never in `dist/`
2. **Documentation** - Keep docs updated in `docs/` folder
3. **Configuration** - Use `src/config.js` for app settings
4. **Assets** - Put static assets in `public/` if not imported
5. **Testing** - Test locally before pushing to production
